## Purpose

Verify generation of Obsidian-compatible Markdown wiki notes.

## Inputs

Canonical knowledge items and relationships generated by the local relationship graph builder.

## Preconditions

Canonical schemas and C8 relationship linking are implemented.

## Steps

1. Generate local wiki Markdown drafts from canonical knowledge items and relationships.
2. Inspect YAML frontmatter.
3. Inspect `[[wiki links]]`.
4. Confirm item-type folder grouping.
5. Confirm open questions and assumptions render when present.
6. Confirm missing relationship endpoints fail clearly instead of creating broken links.

## Expected Output

- Markdown notes generate for supported item types.
- Required frontmatter fields exist.
- Wiki links are valid by item ID/slug.
- Files are grouped by item type.
- Relative file paths are planned for wiki folders.
- Open questions and assumptions render in readable sections.
- Broken relationship endpoints raise a clear render error.

## Real Output

Passed with a local deterministic wiki Markdown renderer:

- Generated `WikiNoteDraft` records with item ID, item type, slug, folder, file name, relative path, Markdown, and linked item IDs.
- Requirement note path example: `requirements/REQ-001-warehouse-stock-movement.md`.
- Source note path example: `sources/SRC-001-source-src-001.md`.
- Markdown includes YAML frontmatter, title, summary, related notes, sources, open questions, and assumptions.
- Wiki links use the deterministic `[[ITEM-ID-slug]]` pattern.
- Renderer rejects relationships that point to missing knowledge items.
